Person Service

Description References Operations Capabilities

Description

Create, update, and find people. Assign person identifiers and manage resolving duplicate identities. Resolving duplicate identities includes the ability to "merge" records.
This would allow for the reading or updating of basic person information. Manipulation of information regarding relationships, academic information, etc. should be handled through the services of record for that information. Generalized information from those areas may be available through this service.

Notes

Assumed that

  • the service exposes attribute sets directly at this level
  • the service may know about more personTypes than it can create people for
  • attributes aren't created for a given person directly through the service (so treated as if created when person is)
  • updates happen on a single attribute set at a time Update: update expects just the changes.
  • deletes occur at the person level
  • person information may not be complete or valid after a personType addition or change Update: reversed. person information must be complete and valid before an addition/change of personType is allowed
  • person deletion may indicate that the identifier is retired at first, so the identifier may be found, but no longer valid. While we talked about the error and look-up as a group, we hadn't come to a consensus on if we should have functionality like this exposed here. As a result, all the DISABLED_IDENTIFIER errors and the fetchReplacementPersonId operation might go away.
  • isValidPersonInfoForPersonType is not listed since validatePersonInfoForPersonType has similar functionality.

Currently sidestepped

  • where the required/updatable designation lies for a set of attributes. Is it with the attribute set? Attached to the connection between personType and attribute set? Changed from having an operation in the setup to one that was person centric to get around that. Solidifying the location would help with being able to specify person-agnostic calls in the setup section (retrieve metadata to be able to recognize which attribute sets to ask for).
    • removed isUpdateablePersonAttributeSetType operation
